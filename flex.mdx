# Flex는 알아야쥐~

CSS에서 자식요소의 정렬을 위해 우리는 관습적이다 싶이 display: flex 속성을 사용합니다.

하지만, 간혹 사용하다보면 "너무 남용하고 있는건 아닐까?"라는 생각이 들 정도입니다.

그래서 display: flex를 몇 개 지워보면 디자인 상에 변화가 없다는 것을 확인하게 되고, 역시나 flex를 남발하고 있었음을 확인할 수 있었습니다.

그래서 이번 기회에 분명 다 알고 사용한다고 생각했던 display: flex를 다시 살펴보는 시간을 가져보고자 합니다.

## Flex란?

MDN 문서에 적힌 설명[3]을 인용하면 다음과 같습니다.

> **CSS Flexible Box Layout**은 사용자 인터페이스 디자인과 단방향 레이아웃에 최적화된 CSS 모듈입니다.
>
> 플렉스 레이아웃 모델에서는, 플렉스 컨테이너의 자식을 어떤 방향으로도 배치할 수 있으며, 자식의 크기도 유연하게("플렉시블") 빈 공간을 채우거나, 컨테이너를 넘어가지 않도록 줄어듭니다.
>
> <u>자식 간의 수평 및 수직 정렬 또한 쉽게 조작할 수 있습니다.</u>

위의 인용글만으로는 한번에 이해하기가 힘들다.

조금 더 쉽게 설명한 Flex & Grid 페이지의 내용[1]을 살펴보도록 하겠습니다.

> Flex는 부모 요소(컨테이너) 아래에서 자식 요소(아이템)들이 한 방향으로 배치가 되도록 합니다.
>
> 따라서 **행 또는 열** 중 한 방향으로 레이아웃을 배치해야 하는 경우에 사용됩니다.
>
> 또한, 다양한 속성을 통해 자식 요소 사이를 일정한 간격으로 떨어지게 할 수도 있고, 정렬을 시키기에 매우 편리합니다.

즉, Flex는 "부모 요소에게 style을 주면, 자식 요소를 행 또는 열의 특정 방향으로 정렬시킬 수 있다." 정도로 요약할 수 있겠습니다.
